## 🧠 What is Clash Dev Analyser?

**Clash Dev Analyser** is a deterministic, AI-free GitHub intelligence tool that gives you real, objective metrics about any GitHub developer or repository.

Forget seniority scores generated by LLMs that hallucinate. This tool reads raw GitHub data — languages, stars, forks, commit history, CI pipelines, README quality, and more — and runs it through a pure algorithmic engine to give you **reproducible, consistent results every time**.

### 🎯 Four Modes

| Mode | What it does |
|------|-------------|
| 🧑‍💻 **Analyze Developer** | Full profile: Dev IQ, language distribution, track detection, role fit scores, roadmap |
| 📦 **Analyze Repo** | Codebase health score, maturity analysis, architecture signals, tooling check |
| ⚔️ **Dev Battle** | Head-to-head comparison of two GitHub developers with a declared winner |
| 🥊 **Repo Battle** | Side-by-side comparison of two repositories |

## ⚙️ Backend

This frontend talks to a **Cloudflare Worker** backend that handles all GitHub API calls, caching, and analysis logic.

> 🔗 **Backend Repo:** [github.com/ajisth69/clash-dev-analyser](https://github.com/ajisth69/clash-dev-analyser/tree/backend)

The mono-repo has two branches:
- [`backend`](https://github.com/ajisth69/clash-dev-analyser/tree/backend) → Cloudflare Worker (TypeScript)
- [`frontend`](https://github.com/ajisth69/clash-dev-analyser/tree/frontend) → Same frontend code, alternative branch

### Backend API Endpoints

| Method | Endpoint | Body |
|--------|----------|------|
| `POST` | `/api/analyze` | `{ "username": "ajisth69" }` |
| `POST` | `/api/analyze-repo` | `{ "repo": "owner/repo" }` |
| `POST` | `/api/compare-devs` | `{ "dev1": "user1", "dev2": "user2" }` |
| `POST` | `/api/compare-repos` | `{ "repo1": "o/r1", "repo2": "o/r2" }` |

## 🚀 Run Locally

### 1. Clone the repo

```bash
git clone https://github.com/ajisth69/dev-analyzer.git
cd dev-analyzer
```

### 2. Install dependencies

```bash
npm install
```

### 3. Start the dev server

```bash
npm run dev
```

The app will be live at `http://localhost:5173`. Make sure you also have the [backend worker](https://github.com/ajisth69/clash-dev-analyser/tree/backend) running locally or deployed to Cloudflare.

### 4. Build for production

```bash
npm run build
```
